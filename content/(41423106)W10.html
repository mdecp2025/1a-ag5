層級化連動結構： 利用 HTML 的嵌套特性（如大臂包含小臂），將機器人的「運動鏈」簡化。當大臂旋轉時，子層的小臂會自動同步位移，省去了繁瑣的三角函數空間座標計算。

物理支點的精確定義： 透過 CSS 的 transform-origin 屬性將旋轉軸心定在關節處（如底部中心）。這是讓方塊從「網頁圖形」轉變為「機械關節」的關鍵，決定了擺動是否符合物理邏輯。

馬達慣性的視覺模擬： 藉由 transition 動畫屬性，能以極簡的代碼模擬出機械馬達啟動與停止時的緩衝與重量感，讓機器人在瀏覽器中的動作從瞬間跳變轉為平滑流暢。

事件驅動與即時回饋： 在 Brython 環境中透過 bind 連結滑桿或按鈕，實現了低延遲的互動控制。這讓我們體會到，瀏覽器不僅能展示靜態網頁，更能作為即時測試機器人邏輯的模擬器。

程式邏輯與物理表現的解耦： Python 負責處理「角度是多少」的邏輯判斷，而 CSS 負責「看起來怎麼動」的物理表現。這種設計模式讓開發者能更專注於機器人的巡邏路徑或自動化演算法的撰寫。
